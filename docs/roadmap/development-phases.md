# Development Phases

## Phase 1 — Repository foundation

Create the monorepo layout, documentation, working rules, environment template, and security baseline. No framework, database, container service, or product feature belongs here.

## Phase 2 — Application scaffolding

Create approved web, extension, backend, and worker foundations with strict typing, linting, and tests.

## Phase 3 — Capture and ingestion

Implement user-selected board capture, change detection, secure ingestion, and lifecycle tracking without live analysis.

## Phase 4 — Secure analysis pipeline

Add vision processing, backend-only Stockfish, durable queues, locked persistence, migrations, and audit trails.

## Phase 5 — Post-game reports

Release charts, critical moments, classifications, comparisons, alternative variations, and summaries only after verified completion.

## Phase 6 — Hardening and release preparation

Expand testing, accessibility, privacy, observability, performance, documentation, and deployment readiness.

## Sequencing rule

Each phase is gated; later work must not be implemented merely because it is documented here.
